---
title: LoadingCache
description: A cache that automatically loads values using a CacheLoader when keys are requested
---

The `LoadingCache` interface extends [Cache](/api/cache) to provide automatic value loading. Values are automatically loaded by the cache using the provided `CacheLoader`, and stored until evicted or manually invalidated.

## Overview

`LoadingCache` provides a simplified access pattern where you can request values by key without providing a loading function each time. The cache uses the `CacheLoader` provided during construction to automatically load missing values.

```java
LoadingCache<String, User> cache = Caffeine.newBuilder()
    .maximumSize(10_000)
    .expireAfterWrite(Duration.ofMinutes(10))
    .build(userId -> database.getUserById(userId));

// Simple access - automatically loads if not present
User user = cache.get("user123");
```

## Automatic Loading Methods

<ParamField path="get(K key)" type="V">
  Returns the value associated with the key, obtaining that value from `CacheLoader.load()` if necessary. If another thread is currently loading the value, this thread waits for that thread to finish.
  
  <Expandable title="parameters">
    <ResponseField name="key" type="K" required>
      The key with which the specified value is to be associated
    </ResponseField>
  </Expandable>
  
  <Expandable title="returns">
    <ResponseField name="value" type="V">
      The current (existing or computed) value associated with the key
    </ResponseField>
  </Expandable>
  
  <Expandable title="throws">
    - `NullPointerException` - if the specified key is null
    - `IllegalStateException` - if the computation detectably attempts a recursive update
    - `CompletionException` - if a checked exception was thrown while loading the value
    - `RuntimeException` or `Error` - if the CacheLoader throws, leaving the mapping unestablished
  </Expandable>
  
  ```java
  LoadingCache<String, User> cache = Caffeine.newBuilder()
      .maximumSize(10_000)
      .build(userId -> {
          System.out.println("Loading user: " + userId);
          return database.getUserById(userId);
      });
  
  // First access - loads from database
  User user1 = cache.get("user123");
  
  // Second access - returns cached value
  User user2 = cache.get("user123");
  ```
</ParamField>

<ParamField path="getAll(Iterable<K> keys)" type="Map<K, V>">
  Returns a map of the values associated with the keys, creating or retrieving those values if necessary. The returned map contains both already cached entries and newly loaded entries.
  
  <Expandable title="parameters">
    <ResponseField name="keys" type="Iterable<K>" required>
      The keys whose associated values are to be returned
    </ResponseField>
  </Expandable>
  
  <Expandable title="returns">
    <ResponseField name="map" type="Map<K, V>">
      An unmodifiable mapping of keys to values for the specified keys
    </ResponseField>
  </Expandable>
  
  <Expandable title="throws">
    - `NullPointerException` - if the collection is null or contains a null element
    - `CompletionException` - if a checked exception was thrown while loading values
    - `RuntimeException` or `Error` - if the CacheLoader throws, if loadAll returns null, or returns a map containing null keys or values
  </Expandable>
  
  ```java
  LoadingCache<String, User> cache = Caffeine.newBuilder()
      .maximumSize(10_000)
      .build(new CacheLoader<String, User>() {
          @Override
          public User load(String key) {
              return database.getUserById(key);
          }
          
          @Override
          public Map<String, User> loadAll(Set<String> keys) {
              // Optimized bulk load
              return database.getUsersByIds(keys);
          }
      });
  
  List<String> userIds = Arrays.asList("user1", "user2", "user3");
  Map<String, User> users = cache.getAll(userIds);
  
  System.out.println("Loaded " + users.size() + " users");
  ```
</ParamField>

## Refresh Methods

<ParamField path="refresh(K key)" type="CompletableFuture<V>">
  Loads a new value for the key, asynchronously. While the new value is loading, the previous value (if any) continues to be returned by `get()` unless it is evicted.
  
  <Expandable title="parameters">
    <ResponseField name="key" type="K" required>
      The key with which a value may be associated
    </ResponseField>
  </Expandable>
  
  <Expandable title="returns">
    <ResponseField name="future" type="CompletableFuture<V>">
      The future that is loading the value
    </ResponseField>
  </Expandable>
  
  <Expandable title="throws">
    - `NullPointerException` - if the specified key is null
  </Expandable>
  
  <Expandable title="info">
    Loading is asynchronous using `AsyncCacheLoader.asyncReload()` if a value exists, or `AsyncCacheLoader.asyncLoad()` otherwise. Returns an existing future without doing anything if another thread is currently loading the value.
  </Expandable>
  
  ```java
  LoadingCache<String, Config> cache = Caffeine.newBuilder()
      .refreshAfterWrite(Duration.ofMinutes(1))
      .build(key -> loadConfig(key));
  
  // Get the current value
  Config config = cache.get("appConfig");
  
  // Manually trigger a refresh
  CompletableFuture<Config> future = cache.refresh("appConfig");
  future.thenAccept(newConfig -> {
      System.out.println("Config refreshed");
  });
  
  // Old value is still returned until refresh completes
  Config stillOldConfig = cache.get("appConfig");
  ```
</ParamField>

<ParamField path="refreshAll(Iterable<K> keys)" type="CompletableFuture<Map<K, V>>">
  Loads new values for each key, asynchronously. While new values are loading, previous values continue to be returned unless evicted. Returns a future containing the loaded values.
  
  <Expandable title="parameters">
    <ResponseField name="keys" type="Iterable<K>" required>
      The keys whose associated values are to be returned
    </ResponseField>
  </Expandable>
  
  <Expandable title="returns">
    <ResponseField name="future" type="CompletableFuture<Map<K, V>>">
      A future containing an unmodifiable mapping of keys to values that are loading
    </ResponseField>
  </Expandable>
  
  <Expandable title="throws">
    - `NullPointerException` - if the specified collection is null or contains a null element
  </Expandable>
  
  ```java
  LoadingCache<String, Product> cache = Caffeine.newBuilder()
      .refreshAfterWrite(Duration.ofMinutes(5))
      .build(productId -> loadProduct(productId));
  
  List<String> productIds = Arrays.asList("prod1", "prod2", "prod3");
  
  // Trigger refresh for multiple keys
  CompletableFuture<Map<String, Product>> future = cache.refreshAll(productIds);
  
  future.thenAccept(products -> {
      System.out.println("Refreshed " + products.size() + " products");
  });
  ```
</ParamField>

## Inherited Methods

`LoadingCache` inherits all methods from [Cache](/api/cache), including:

- `getIfPresent(K key)` - Get without loading
- `get(K key, Function<K, V> mappingFunction)` - Get with custom loader
- `getAllPresent(Iterable<K> keys)` - Get multiple without loading
- `put(K key, V value)` - Manual insertion
- `putAll(Map<K, V> map)` - Bulk insertion
- `invalidate(K key)` - Remove single entry
- `invalidateAll(Iterable<K> keys)` - Remove multiple entries
- `invalidateAll()` - Clear cache
- `estimatedSize()` - Get approximate size
- `stats()` - Get statistics
- `asMap()` - Get Map view
- `cleanUp()` - Perform maintenance
- `policy()` - Access policy controls

## Usage Examples

### Basic Loading Cache

```java
LoadingCache<String, User> userCache = Caffeine.newBuilder()
    .maximumSize(10_000)
    .expireAfterWrite(Duration.ofMinutes(10))
    .build(userId -> {
        System.out.println("Loading user: " + userId);
        return database.getUserById(userId);
    });

// Simple access - automatically loads if not cached
User user = userCache.get("user123");

// Bulk access
List<String> userIds = Arrays.asList("user1", "user2", "user3");
Map<String, User> users = userCache.getAll(userIds);
```

### With Bulk Loading

```java
LoadingCache<String, Product> productCache = Caffeine.newBuilder()
    .maximumSize(50_000)
    .expireAfterWrite(Duration.ofMinutes(30))
    .build(new CacheLoader<String, Product>() {
        @Override
        public Product load(String productId) {
            // Single load fallback
            return database.getProduct(productId);
        }
        
        @Override
        public Map<String, Product> loadAll(Set<String> productIds) {
            // Optimized batch load from database
            return database.getProducts(productIds);
        }
    });

// Efficient bulk loading
List<String> productIds = getProductIdsFromRequest();
Map<String, Product> products = productCache.getAll(productIds);
```

### With Automatic Refresh

```java
LoadingCache<String, StockPrice> priceCache = Caffeine.newBuilder()
    .maximumSize(1000)
    .refreshAfterWrite(Duration.ofSeconds(30))
    .build(symbol -> stockService.getCurrentPrice(symbol));

// First request loads the price
StockPrice price1 = priceCache.get("AAPL");

// After 30 seconds, next request triggers async refresh
// Old price is returned immediately, new price loads in background
StockPrice price2 = priceCache.get("AAPL");

// Manual refresh when needed
priceCache.refresh("AAPL");
```

### With Error Handling

```java
LoadingCache<String, Config> configCache = Caffeine.newBuilder()
    .maximumSize(100)
    .expireAfterWrite(Duration.ofMinutes(10))
    .build(configKey -> {
        try {
            return configService.loadConfig(configKey);
        } catch (ConfigNotFoundException e) {
            // Return default config instead of throwing
            return Config.getDefault(configKey);
        }
    });

try {
    Config config = configCache.get("app.settings");
} catch (CompletionException e) {
    // Handle checked exceptions wrapped in CompletionException
    if (e.getCause() instanceof IOException) {
        // Handle I/O error
    }
}
```

### With Reload Support

```java
LoadingCache<String, Document> documentCache = Caffeine.newBuilder()
    .maximumSize(500)
    .refreshAfterWrite(Duration.ofMinutes(5))
    .build(new CacheLoader<String, Document>() {
        @Override
        public Document load(String docId) throws Exception {
            return documentService.loadDocument(docId);
        }
        
        @Override
        public Document reload(String docId, Document oldDoc) throws Exception {
            // Only reload if document has been modified
            if (documentService.isModified(docId, oldDoc.getVersion())) {
                return documentService.loadDocument(docId);
            }
            return oldDoc;
        }
    });

// Get document (loads if not cached)
Document doc = documentCache.get("doc123");

// After 5 minutes, automatic refresh uses reload() method
// which can optimize by checking if document changed
Thread.sleep(Duration.ofMinutes(6).toMillis());
Document refreshedDoc = documentCache.get("doc123");
```

### With Statistics

```java
LoadingCache<String, Report> reportCache = Caffeine.newBuilder()
    .maximumSize(100)
    .recordStats()
    .build(reportId -> generateReport(reportId));

// Use the cache
for (String reportId : requestedReports) {
    Report report = reportCache.get(reportId);
}

// Analyze performance
CacheStats stats = reportCache.stats();
System.out.printf("Load count: %d%n", stats.loadCount());
System.out.printf("Load success: %d%n", stats.loadSuccessCount());
System.out.printf("Load failure: %d%n", stats.loadFailureCount());
System.out.printf("Average load time: %.2f ms%n", 
    stats.averageLoadPenalty() / 1_000_000.0);
```
